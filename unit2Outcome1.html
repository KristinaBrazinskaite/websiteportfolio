1.5 Describing copyright and other constraints affecting websites

Website Copyrights

Every original piece of work must be protected by copyrights. What is the copyright of a website? "Copyright protects original works of authorship, including the text, graphics, photographs, sound recordings and audio, visual elements of websites and their underlying computer programs." [1]. It is essential to have copyrights for authentic work. If somebody decides to use the content of your website without your consent, it is classified as data breaching. Nobody can "copyright a domain name or the title of a website and copyright ideas, procedures, systems or methods of operation" [1]. If we have website copyrights for our website, we should include our website application terms and conditions [2]. The advantage of having those copyrights, "would recover statutory damages and attorneys fees" [1].

Other Constraints Affecting Websites

One of the website constraints could be "website viewports" [3], meaning that page has to be responsive: opened on different web browsers, on different devices, for example, website and mobile application. Users should be able to zoom in and out some parts of your website. It increases readability, helps to navigate especially maps should be designed in a way that users would have a smooth experience. Pages should have relevant content. For example, if a child is visiting your website and you are selling alcohol there, then should be some restrictions that would block the viewer from visiting this site, maybe some alerts asking about your age. Another restriction could be "security measures up to date" [4]. For example, if anyone purchasing items online, then their data should be secured and safe, the details should be protected from public access.

1.7 Describe which are the appropriate ﬁletypes for websites

HTML File

We are already using many different files daily but which are the appropriate ﬁletypes for websites? I will describe each the most common filetype and where it is used. When I started my website, the first thing I have done, created a .html file in my Visual Studio. This code is called the "HyperText Markup Language file" [1]. It is a language that is used today. The great thing is that once we are running the Html file, we can open our website on Internet Explorer or Google Chrome. Html is designed for creating a website structure, we can write all the text, add links of images or videos or audio or any other filetype. Html uses tags that help to render details in a browser, for example, if we decided to add title, we use h1 tag, each number has a different size for a title, 1 is the biggest and 6 is the smallest. If we want to create some text inside our website, we can use p tag for a paragraph. Another use of Html is to navigate from one page to another, I used a reference tag to reference other Html pages or existing website links to my portfolio for example. Html is responsive because it can work on different platforms like mobile devices, tablets, websites [2], there is new version Html5 which has more features for "animation, games" [1].

CSS File

CSS means "Cascading StyleSheet and describes the style of an HTML document " [3]. I used CSS for designing text, adding different font or change font size, adding background color on my website. It is a designing tool and let you write classes, where you can manipulate image size, location. On a website page, we specify borders, margins, padding. It is some space around an object like image, text, buttons, etc.

JavaScript File

JavaScript is "is a text-based programming language used both on the client-side and server-side" [4], mainly used for website functionality. For example, if we want certain buttons to work or create image sliding show when each image is displayed one after each other, we have to write JavaScript functionality. This would be an example of front end website development. Where would we use JavaScript for the back end? We have not covered this with Code Nation yet but from my own experience with different programming languages like Java or C#, I could say that we have to create databases first, which we would connect with for example JavaScript CRUD functionality. CRUD means create, read, update and delete information on a website. Where else we can use JavaScript for the back end, is when we have to login into a website or fil in registering form online, shopping online because we have to make payments. All required data stays on a website but it is saved in databases that are protected by cybersecurity.

Images, Videos, Audio, Document Files

"Multimedia comes in many different formats like images, music, sound, videos, records, films, animations." [5]. If we want to display an image on our website, we should use a photograph file format of .jpeg, .jpg or.png. I used .jpg. Other appropriate filetypes for websites videos are .mp4 or .ogg [6] and for audio .mp3 or.ogg [7], and for youtube videos, we could simply insert youtube link into our Html file where appropriate. I am interested in using .pdf filetype because "Portable Document Format" [2] is convenient files are looking the same without any changes. "Adobe Acrobat files were invented so that documents could be transferred between computers and indeed platforms, and still look the same" [2].
