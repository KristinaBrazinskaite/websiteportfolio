3.2 Describe ways to improve productivity and efficiency

Productivity and Efficiency Meaning

How to be more productive and efficient. There are many ways but first, l would like to describe what productivity and efficiency mean. "Productivity and efficiency are two of the key goals of any business enterprise. But they are two very different things and often compete with each other. Improved productivity can come at the expense of efficiency and improved efficiency can reduce productivity. Put simply, productivity is the quantity of work produced by a team, business or individual. Efficiency, on the other hand, refers to the resources used to produce that work. So, the more effort, time or raw materials required to do the work, the less efficient the process. Efficiency can also refer to the quality of the output" [1]. As I said before both work together, we could produce a lot of features on a website but half of them not functioning well, then efficiency is reduced and not meeting quality standards.

Ways To Improve Productivity and Efficiency, Planning

To produce more quality, we need to be organised and have a plan in hand. We have to be very specific and know exactly what are we going to create, have a vision of the website, predict how long each task would take. I used Gantt charts before, it is a visual calendar with set dates and time on Microsoft Project. I remember I printed it for my researching project and it worked because every time you have to check your calendar what are you going to do next, you spending time on it, it is easier if you have everything in one page graphically represented. For our website we used Trello boards, which are digital and easy to use, you can have access on your phone with all data.

3.4 Describe how you would go about testing digital solutions

Testing Websites

Testing is a piece of work that has to be accomplished in the software development cycle. The software development life cycle is made of few sections: planning, analysis, design, implementation, testing and integration, maintenance. We have not used JavaScript on our website so we can not use testing tools like Selenium or FireUnit. We created a website with Html and CSS only, so we can test the performance of our website on different web browsers like Internet Explorer and Google Chrome or test our application on different operating systems like Windows and iOS. Sometimes design slightly different on each operating system. When we had a game group project, our first page of the game had some issues with design because a button that meant to appear in the middle of the page looked different on Windows and iOS. I had to change a design by changing style in CSS of button location. Testing is always working together with developing a website because everything we are making, we are testing. It is a good practice to do that because any bigger issues with code could be avoided if the code was tested regularly and more often. If we think in general about testing, it depends on an object-oriented programming language which testing tools to use. When I had my Java project of creating CRUD functionalities, I used Eclipse for my Java as environment and Eclipse normally have free testing tools implemented. I used JUnit to test my Java code. It was showing green lines if the test was successful and red - if failure.

Rules For Testing

Some rules apply to test if people are not sure where to begin. One of them is "functionality, usability, interface, compatibility, performance, security" [1]. I could say I checked functionality for Html and CSS, tested by running a program on the browser. If code was wrong and made changes won't happen. Usability should be tested by external people who can express their opinion if certain features have used and required for the website. The interface was examined many times, for example, image size or different text, checked immediately. To test compatibility is easy just to change screen size from full to partial, imagine it is a mobile device with a smaller screen and how your design would look. Performance there to check if all implemented functionalities are in order and responding by simply clicking on certain links or buttons. Security is the last but not least important because any application we make, we want to be secure to customers, all data should be protected, especially when users having transactions over the website.
